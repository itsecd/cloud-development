# Cloud Development Labs

Выполнил: Панкеев Глеб, 6512  
Вариант: №30, объект «Кредитная заявка»  
Алгоритм балансировки: Query Based

## Что реализовано

### Лабораторная 1. Кэширование

- API генерации кредитной заявки на базе `Bogus`
- Кэш заявок через `SQS` (LocalStack)
- Структурное логирование в сервисе
- Оркестрация через `.NET Aspire`

Endpoint:

- `GET /api/creditapplication?id={id}`

### Лабораторная 2. Балансировка нагрузки

- Оркестрация запуска 3 реплик API:
  - `projectapp-api-r1` (`7001`)
  - `projectapp-api-r2` (`7002`)
  - `projectapp-api-r3` (`7003`)
- API Gateway на `Ocelot`:
  - `projectapp-gateway` (`7000`)
- Кастомный балансировщик `QueryBased`:
  - выбор реплики по формуле `id % N`, где `N` — число реплик

## Модель «Кредитная заявка»

1. Идентификатор в системе — `int`
2. Тип кредита — `string`
3. Запрашиваемая сумма — `decimal`
4. Срок в месяцах — `int`
5. Процентная ставка — `double`
6. Дата подачи — `DateOnly`
7. Необходимость страховки — `bool`
8. Статус заявки — `string`
9. Дата решения — `DateOnly?`
10. Одобренная сумма — `decimal?`

## Правила генерации

- Тип кредита выбирается из справочника (`Потребительский`, `Ипотека`, `Автокредит` и др.)
- Процентная ставка округляется до 2 знаков и не ниже ключевой ставки ЦБ (`CreditApplicationSettings:CentralBankKeyRate`)
- Запрашиваемая и одобренная суммы округляются до 2 знаков
- Одобренная сумма не превышает запрашиваемую
- Дата подачи берётся не более чем за 2 года от текущей даты
- Дата решения задаётся только для терминальных статусов (`Одобрена`, `Отклонена`)
- Для статуса `Одобрена` обязательно есть одобренная сумма
- Для остальных статусов одобренная сумма отсутствует

## Запуск

1. Запустить AppHost (`ProjectApp.AppHost`)
2. Открыть Swagger API или клиент `Client.Wasm`
3. Запросы через gateway:
   - `http://localhost:7000/api/creditapplication?id=32`
   - `http://localhost:7000/api/creditapplication?id=654`
   - `http://localhost:7000/api/creditapplication?id=55`
