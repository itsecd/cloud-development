# Лабораторная работа №2: «Балансировка нагрузки» — реализация API-гейтвея и настройка его работы
# Вариант №5: Сотрудник компании
Реализация микросервиса генерации сотрудника компании с кэшированием и оркестрацией на базе .NET Aspire.

# Цель
Реализация проекта микросервисного бекенда.

# Задачи:

1 - Настроить оркестрацию на запуск нескольких реплик сервиса генерации,
2 - Реализовать API-гейтвей на основе Ocelot,
3 - Внедрить алгоритм балансировки нагрузки в соответствии с вариантом.

# Комментарии к характеристикам генерации:
1 - ФИО - конкатенация фамилии, имени  и отчества через пробел, все из раздела Name

2 - Должность выбирается из двух заранее подготовленных справочников -  справочника профессий (“Developer”, “Manager”, “Analyst” и т.д.) и суффиксов к ним (“Junior”, “Middle”, “Senior” и т.д.)

3 - Отдел берется из раздела Commerce

4 - Дата приема берется не более, чем 10 лет назад от текущей даты

5 - Генерация суммы оклада должна коррелировать с суффиксом должности и не иметь более 2 знаков в дробной части

6 - Электронная почта генерируется из раздела Internet

7 - При отсутствии индикатора увольнения дата увольнения не заполняется

8 - Телефонный номер должен браться из раздела Phone и быть формата +7(***)***-**-**
